The API Integration Problem

Stop reinventing the wheel for every API integration

Most software engineers consider infrastructure components like Kafka, NATS, DB, ETCD as the "Wheel" and rush to implement them. However, the real challenge isn't the infrastructure—it's the API integrations and maintenance.

Using infrastructure components takes weeks or months, but building stable APIs and integrations can take years and enormous amounts of money with constant costly maintenance.

GSQL solves this by providing a single, universal API for querying any Go struct's graph model at runtime.

Features

Powerful capabilities for modern applications

SQL-like Syntax

Familiar SQL syntax that developers already know, with support for SELECT, WHERE, ORDER BY, and more.

Graph Model Support

Native support for complex nested structures, arrays, maps, and deep object relationships.

Runtime Introspection

Automatic model discovery and schema generation using Go's reflection system.

Advanced Filtering

Powerful filtering with pattern matching, comparison operators, and logical expressions.

Sorting & Pagination

Built-in support for sorting, pagination, and result limiting with intuitive syntax.

Type Safety

Strong typing with Go's type system ensures compile-time safety and runtime reliability.

Architecture

Four core components working together

Introspector

Accepts Go structs and introspects them to discover attributes and sub-objects, creating internal and graph schemas.

Parser

Parses query strings, validates syntax, and divides queries into columns, tables, and criteria components.

Interpreter

Takes parsed queries, validates them against the schema, and creates executable query instances for filtering.

Instance System

Provides string representations for instances and attributes with seamless getter/setter functionality.

Quick Start

Get up and running in minutes

Installation

go get github.com/saichler/gsql/go

Basic Usage

package main

import (
    "fmt"
    "github.com/saichler/gsql/go/gsql/interpreter"
)

// Define your model
type Employee struct {
    Name      string
    Age       int32
    Addresses []Address
}

type Address struct {
    Line1   string
    Line2   string
    Zip     string
    Country string
}

func main() {
    // Create a query
    query, err := interpreter.NewQuery(
        "select name, age from employee where name='John' or age>25", 
        resources, // your IResources implementation
    )
    if err != nil {
        panic(err)
    }
    
    // Test with your data
    employees := []*Employee{
        {Name: "John", Age: 30},
        {Name: "Jane", Age: 25},
        {Name: "Bob", Age: 35},
    }
    
    // Filter matching elements
    for _, emp := range employees {
        if query.Match(emp) {
            fmt.Printf("Matched: %+v\n", emp)
        }
    }
}

Query Examples

See GSQL in action

Basic Queries

-- Select all fields
select * from employee

-- Select specific fields
select name, age from employee where age > 25

-- Complex conditions
select name from employee 
where (age > 25 and name = 'John') or country = 'USA'

Deep Path Navigation

-- Query nested objects
select name from employee 
where addresses.country = 'USA'

-- Query array elements
select name from employee 
where addresses[0].zip = '12345'

-- Query map values
select name from employee 
where metadata.department = 'Engineering'

Advanced Features

-- Sorting and pagination
select * from employee 
where age > 25 
sort-by age descending 
limit 10 page 1

-- Case-sensitive matching
select * from employee 
where name = 'john' match-case

-- Pattern matching
select * from employee 
where name = 'J*'

API Usage

// Create and use queries
query, _ := interpreter.NewQuery(
    "select * from employee where age > 25", 
    resources
)

// Test individual objects
if query.Match(employee) {
    fmt.Println("Employee matches!")
}

// Filter collections
filtered := query.Filter(employees, true)

// Get query properties
properties := query.Properties()
criteria := query.Criteria()
